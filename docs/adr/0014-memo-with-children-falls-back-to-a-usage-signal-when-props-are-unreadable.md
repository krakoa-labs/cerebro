# Memo with children falls back to a usage signal when props are unreadable

ADR-0013 made the Memo-with-children indicator fire only on the provably-inert shape: a comparator-free `memo()` whose props declare statically-readable element-typed `children`. That precision left a recall gap. A Component whose props arrive through a form the source reader cannot resolve — a polymorphic cast (`const C = forwardRef(…) as ForwardRefComponent<"li", Props>`, where the contract lives in the cast's type argument) or a utility/mapped type (`type Props = Override<HTMLAttributes, BaseProps>`, where `children` is nested behind a type operator) — was left unflagged even when its `children` were genuinely element-typed. On a real 123-Component design system the indicator caught 6 of ~17 real cases; the misses were dominated by these two unreadable-props shapes, both pervasive in polymorphic-ref component conventions.

Cerebro now keeps the readable-type rule authoritative and adds a usage fallback for exactly the blind spots. When the props contract resolves to a member list, that reading is final: the indicator flags only element-typed `children` and trusts a primitive or absent `children` member as a genuine non-footgun, preserving every exclusion ADR-0013 earned. When the members cannot be resolved from source, the indicator falls back to a structural usage signal — whether the Component destructures a `children` binding out of its props parameter (`({ children, … }) => …`). A Component that names `children` among the props it pulls apart accepts children, the shape that defeats the memo, regardless of how the type is spelled. The fallback fires only in the unreadable case, so it never overrides a type the reader can see.

This deliberately relaxes ADR-0013's "provably inert" stance into "inert on the available evidence", trading a sliver of soundness for recall. The trade is bounded: the readable path is unchanged, the comparator exclusion still applies, and the fallback reaches only Components whose props were already unreadable — where the prior behavior was to guess `false` anyway (the quiet direction, ADR-0005). The new soundness cost is a primitively-typed `children` hidden behind an unreadable type *and* destructured in the parameter, which would be a false positive; on the validating design system that set was empty.

## Considered Options

Improving the type reader to resolve the missed shapes precisely was the soundness-preserving alternative. Reading the last type argument of a component cast would recover the polymorphic-cast case, but it keys on a convention — `ForwardRefComponent`/`MemoComponent` are local to one design system — which ADR-0011 already rejected as non-generic, and it does nothing for the utility-type case. Evaluating utility and mapped types (`Override<…>`, `interface extends`, intersections) to derive `children` would be generic, but it reimplements a type checker over an AST the parser exposes only structurally, and it remains partial against arbitrary mapped types. Both options buy back precision the validating set never needed, at a cost out of proportion to the gap; they are deferred until a concrete case demands them.

A pure usage rule — flag any comparator-free `memo` that destructures `children`, ignoring the props type entirely — was considered and rejected. It is simpler to state, but it discards the readable-type exclusions (a readable `children: string` that is destructured would become a false positive), spending soundness the reader is perfectly able to preserve. Gating the fallback on unreadable members keeps the precise verdict wherever the contract can be read and relaxes only where it cannot.

The fallback signal is scoped to destructuring in the props parameter; body-level destructuring (`const { children } = props`) and member access (`props.children`) are not consulted. Every observed case destructures in the parameter, and the omitted forms leave a Component unflagged — the quiet direction — rather than risk a wider false-positive surface.
